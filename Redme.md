## 🚀 Key Features & SQL Concepts
This system handles ticket management through three core tables: `Users`, `Matches`, and `Bookings`. The SQL queries implemented include:
*   **Pattern Matching:** Using `LIKE` and `ILIKE` for user searches.
*   **Null Handling:** Using `IS NULL` and `COALESCE` for status management.
*   **Joins:** Using `INNER JOIN` and `LEFT JOIN` to link users, matches, and bookings.
*   **Aggregations & Subqueries:** Solving complex filtering requirements using `AVG()` and nested queries.
*   **Pagination:** Using `LIMIT` and `OFFSET` to display specific data subsets.
